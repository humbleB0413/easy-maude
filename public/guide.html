Easy MAUDE Search Guide

Complete reference for searching the FDA MAUDE adverse event database.

What is the MAUDE Database?

MAUDE (Manufacturer and User Facility Device Experience) is the FDA's database of Medical Device Reports (MDRs). It contains adverse event reports submitted to the FDA by:

  • Device manufacturers — required to report deaths, serious injuries, and malfunctions
  • Device importers — required to report deaths and serious injuries
  • Device user facilities (hospitals, nursing homes, etc.) — required to report deaths and serious injuries
  • Voluntary reporters — patients, healthcare professionals, and consumers

MAUDE is a critical resource for post-market surveillance, regulatory submissions, safety signal detection, and competitor intelligence. The database contains millions of reports dating back to 1992.

Note: MAUDE reports are submitted by the listed reporter. FDA has not conducted independent verification of all data. Reports should be interpreted in the context of reporting practices and device use rates.

Getting Started

Easy MAUDE uses a simple tag:value syntax. You don't need to know OpenFDA's field names — just use intuitive tags like brand, code, or event.

Open the search page and type your query in the search box. Results will appear as soon as the data is retrieved from the FDA API.

Tip: If you don't specify a date, Easy MAUDE automatically searches the last 12 months. For broader searches, add an explicit date range.

Basic Search Syntax

The fundamental building block is a tag:value pair. Multiple pairs are combined with AND by default:

brand:dexcom event:Malfunction → brand is dexcom AND event type is Malfunction

Phrase Matching

Multi-word values are automatically quoted for you — just type them naturally:

manufacturer:boston scientific → searches for "boston scientific" (as a phrase)

Case Sensitivity

Most searches are case-insensitive. Exception: code and udi tags are automatically uppercased (FDA product codes use uppercase identifiers).

code:frn → automatically becomes code:FRN

Tag Reference

These are all supported search tags and their corresponding OpenFDA database fields:

Tag What it searches Example
brand Commercial brand name of the device (device.brand_name) brand:dexcom
generic Generic or technical device name (device.generic_name) generic:catheter
manufacturer Manufacturer name as reported (device.manufacturer_d_name) manufacturer:medtronic
model Device model number (device.model_number) model:G7
code FDA 3-letter product code — auto-uppercased (device.device_report_product_code) code:FRN
udi UDI Device Identifier barcode (device.udi_di) — auto-uppercased udi:00380740006547
event Event type: Malfunction, Injury, Death, Other (event_type) event:Malfunction
date Date FDA received the report, YYYYMMDD format (date_received) date:20240101-20241231
event_date Actual date the adverse event occurred (date_of_event) event_date:20231015-
report MDR report key — unique identifier (mdr_report_key) report:3014449-2024
reporter Reporter occupation code: P=Physician, N=Nurse, H=Hospital, etc. (reporter_occupation_code) reporter:P
text Full-text search in event descriptions and manufacturer narratives (mdr_text.text) text:battery failure
problem Product problem type (e.g. Failure to Operate, Leakage) (product_problems) problem:Leakage

Date Searching

Use the date tag with 8-digit date values (YYYYMMDD). If you omit the date tag entirely, the last 12 months are searched automatically.

Date Range Formats

date:20240101-20241231 → from Jan 1 to Dec 31, 2024 date:20240101- → from Jan 1, 2024 to today date:-20241231 → from the beginning of the database to Dec 31, 2024
Large date ranges: Queries covering many years may retrieve hundreds of thousands of records. Easy MAUDE handles this automatically by splitting the date range into smaller windows and merging results — but large searches can take several minutes.

OR Queries

Use a pipe character | to combine multiple conditions with OR logic:

brand:dexcom | brand:abbott → either Dexcom OR Abbott brand brand:dexcom | brand:abbott | brand:libre → any of three brands

You can mix AND and OR in the same query. OR groups are evaluated first, then AND:

(brand:dexcom | brand:abbott) event:Death → Dexcom or Abbott, with Death event type → in Easy MAUDE syntax (no parentheses needed): brand:dexcom event:Death | brand:abbott event:Death

Practical Search Examples

Competitive Intelligence

Find all malfunctions for a specific CGM brand in the last 2 years
brand:dexcom event:Malfunction date:20230101-20241231
Combines brand filter + event type + explicit date range.
Compare adverse events across multiple insulin pump brands
brand:medtronic | brand:insulet | brand:tandem
OR logic to pull reports for three brands in one search.

Product Code Searches

Search by FDA 3-letter product code
code:FRN
FRN is the product code for continuous glucose monitors (CGMs). All device classes with this code will appear. Codes are automatically uppercased.
Product code + specific event type
code:QBJ event:Death
All death reports for devices with product code QBJ (ventricular assist devices).

Post-Market Surveillance

Full-text search for a specific failure mode
brand:heartware text:thrombosis
Finds reports where the event narrative mentions "thrombosis" for HeartWare devices.
Patient deaths reported by physicians
code:FRN event:Death reporter:P
P = Physician reporter occupation code. Useful for filtering high-quality reports.

Regulatory Submissions

All events for a specific device model
manufacturer:abbott model:FreeStyle
Useful for 510(k) substantial equivalence or PMA safety data compilations.
Long-range search for a legacy device
brand:Bjork-Shiley event:Malfunction date:19920101-20001231
Searches the full MAUDE archive. Large results are handled via automatic date splitting.

Getting an OpenFDA API Key

By default, Easy MAUDE uses anonymous OpenFDA API access, which is limited to 1,000 requests per day across all users of the service. If you perform frequent or large searches, you should obtain a free personal API key:

  1. Visit open.fda.gov/apis/authentication
  2. Enter your email address and click "Get API Key"
  3. Copy the key from the confirmation email
  4. Open Easy MAUDE → Settings (gear icon) → API Configuration → paste your key

With an API key, your daily limit is 120,000 requests — more than enough for any normal use.

Understanding MAUDE Report Fields

Event Types

  • Malfunction — Device failure that did not cause or contribute to a serious patient outcome, but could if it recurred
  • Injury — Device caused or contributed to a serious injury requiring medical or surgical intervention
  • Death — Device caused or contributed to a patient death
  • Other — Does not fall into the above categories

Reporter Occupation Codes

  • P — Physician
  • N — Nurse
  • H — Hospital
  • U — User Facility
  • D — Distributor
  • M — Manufacturer
  • I — Importer
  • C — Consumer
  • O — Other Health Professional

MDR Report Key

Each report has a unique MDR Report Key in the format NNNNNNN-YYYY. Use the report tag to look up a specific report if you know its key from another source.

Frequently Asked Questions

Why does my search show 0 results when I know reports exist?

The most common causes: (1) The date range defaults to the last 12 months — try expanding it with an explicit date: tag. (2) Product code searches require exactly the FDA's 3-letter code — check it at FDA's Product Code Database. (3) Manufacturer names vary widely — try searching by brand or product code instead.

How recent is the MAUDE data?

The OpenFDA MAUDE database is updated weekly. There is typically a 1–2 week lag between a report being submitted and appearing in the database.

Why does a large search take a long time?

OpenFDA limits responses to 1,000 records per request and 26,000 records per date window. For queries that match more records, Easy MAUDE automatically splits the date range into progressively smaller windows and retrieves each one separately. A query returning 500,000 records may make 50–100 API calls and take 2–5 minutes. A progress bar and cancel button are available during the search.

Can I export results to Excel?

Yes. On the search results page, click the Export Excel button to download a .xlsx file containing all retrieved records with all available fields.

Are the results stored?

Yes, all search results are saved locally in your browser's IndexedDB. You can access previous searches from the Search History panel (Settings → Advanced → Search History). Results are automatically cleaned up after 30 days when storage exceeds 200 MB.

Is my data sent to any server?

No. Only your search query (not your identity) is sent to api.fda.gov when fetching data. No personal information is sent anywhere. See the Privacy Policy for full details.
